Interface Things<T>

The Things interface implements for type T the functionality of Something minus cloning, without T actually needing to extend Something.

It also lets type T, which might already be Something, be a different Something.

compare

  • compare(lhs: T, rhs: T): ComparisonResult

  • Inherited from PartialOrder.compare

    • Defined in src/interfaces/comparable.ts:28

    Compares lhs with rhs and returns how they relate to each other, e.g. LESS if lhs is less than rhs.

    It is possible to represent partial orders, by returning UNRELATED if lhs and rhs are not related.

    Comparison must have the following properties for all x, y, z of type T:

    • Compatible with Equality: compare(x, y) is true iff equals(x, y) is true
    • Antisymmetry:
      • compare(x, y) === LESS implies compare(y, x) === GREATER
      • compare(x, y) === GREATER implies compare(y, x) === LESS

    From this it follows that compare(x, y) === UNRELATED implies compare(y, x) === UNRELATED.

equals

  • equals(lhs: T, rhs: T): boolean

  • Inherited from PartialOrder.equals

    • Defined in src/interfaces/equatable.ts:10

    Tests if lhs is equal to rhs. Equality testing must obey the following rules for all x, y, z of type T:

    • Reflexivity: equals(x, x) is true
    • Symmetry: if equals(x, y) is true, then equals(y, x) is true as well
    • Transitivity: if equals(x, y) is true, and equals(y, z) is true, then x.equals(z) is true as well
    • Undefinedness: if equals(undefined, x) or equals(x, undefined) is true, then x === undefined

    • rhs: T

    Returns boolean
