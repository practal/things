Interface MapThings<M, Key, Value>

General interface for working with maps. The interface works for both mutable and immutable maps. Keys must be immutable.

compare

  • compare(x: Readonly<M>, y: Readonly<M>): number

  • Inherited from Things.compare

    • Defined in things.mts:46

    Compares x with y and returns how they relate to each other.

    It is possible to represent partial orders by returning NaN if x and y are not related.

    Comparison must have the following properties for all a, b, c and d in the domain:

    • Compatibility with Equality:
      • If equals(a, b) and equals(c, d) are both true, then a relates to c in the same way as b relates to d.
      • We have that equals(a, b) is true iff compare(a, b) === 0 is true.
      • If a and b are related, then both equals(a, a) and equals(b, b) are both true.
    • Antisymmetry:
      • compare(a, b) < 0 implies compare(b, a) > 0
      • compare(a, b) > 0 implies compare(b, a) < 0
    • Transitivity:
      • If compare(a, b) < 0 and compare(b, c) < 0, then compare(a, c) < 0.

    Parameters

    • x: Readonly<M>
    • y: Readonly<M>

    Returns number

    • 0 if x and y are equal
    • a negative value if x < y,
    • a positive value if x > y
    • NaN if x and y are unrelated

equals

  • equals(x: Readonly<M>, y: Readonly<M>): boolean

  • Inherited from Things.equals

    • Defined in things.mts:22

    Tests if x is equivalent to y where both x and y are assumed to be in the domain. Equivalence testing must be reflexive, symmetric and transitive on the domain:

    • Reflexivity: The condition equals(x, x) holds.
    • Symmetry: If equals(x, y) is true, then so is equals(y, x).
    • Transitivity: If equals(x, y) and equals(y, z) are both true, then so is equals(x, z).

put

  • put(map: M, key: Key, value: Value): { old: undefined | Value; result: M }
    • Defined in map_things.mts:38

    Assigns value to key in map. If the key was previously associated with some value v, v is returned as old, otherwise old is undefined.

    The changed map is returned as result. Note that put works for both immutable and mutable maps: For mutable maps, the result will be just map, that is result === map will hold.
